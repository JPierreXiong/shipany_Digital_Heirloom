## Pourquoi les Méthodes Traditionnelles Échouent pour le Crypto

### Le Problème du Multisig

De nombreux investisseurs crypto s'appuient sur des portefeuilles multisig, croyant que le partage de clés avec des parties de confiance offre la sécurité. Cependant, cette approche présente des défauts critiques :

- **Point de Défaillance Unique** : Si un détenteur de clé devient indisponible, tout le portefeuille devient inaccessible
- **Risque de Sécurité** : Le partage de clés privées augmente la surface d'attaque
- **Pas d'Automatisation** : La récupération manuelle de clés ne tient pas compte des scénarios d'incapacité
- **Complexité Juridique** : Les testaments traditionnels ne peuvent pas légalement transférer les clés privées

### Le Piège du "Écrire sur Papier"

Certains investisseurs écrivent les phrases de récupération ou les clés privées dans des emplacements physiques :

- **Vulnérabilité Physique** : Le papier peut être perdu, endommagé ou volé
- **Pas de Libération Temporelle** : Les clés sont accessibles immédiatement, même si vous êtes encore en vie
- **Pas de Vérification** : Aucun moyen de s'assurer que la bonne personne accède aux actifs au bon moment

### Fragment A/B : Éliminer les Points de Défaillance Uniques

Pour les utilisateurs Pro, Afterglow introduit la **Division Physique Fragment A/B** :

- **Fragment A (Cloud)** : Fragment chiffré stocké sur des serveurs sécurisés—ne peut pas déchiffrer seul
- **Fragment B (Physique)** : Composant de déchiffrement unique livré via courrier physique ShipAny ou détenu par un bénéficiaire désigné—ne peut pas déchiffrer seul

**Logique de Sécurité** : Les deux fragments doivent être combinés pour accéder à votre coffre. Cela reflète les coffres-forts réels—même si le Fragment A est compromis (violation cloud) ou le Fragment B est perdu (vol physique), vos actifs restent sécurisés.

## Plongée Technique : Architecture de Preuve Zéro-Connaissance

### Comment Ça Fonctionne

1. **Chiffrement** : Votre Mot de Passe Principal dérive une clé symétrique via PBKDF2
2. **Fragmentation** : Pour les plans Pro, la clé de déchiffrement est divisée en Fragment A (cloud) et Fragment B (physique)
3. **Stockage** : Seules les données chiffrées sont stockées sur les serveurs Afterglow
4. **Déclenchement** : Le système de battement de cœur détecte l'inactivité et initie le workflow d'héritage
5. **Libération** : Les bénéficiaires s'authentifient en utilisant des jetons de libération et (pour Pro) le Fragment B
6. **Déchiffrement** : Les fragments se combinent pour reconstruire la clé et déchiffrer votre coffre

## FAQ : Héritage Crypto

### Comment savoir que mes clés privées sont sécurisées ?

Afterglow utilise le **chiffrement zéro-connaissance**, ce qui signifie que vos clés sont chiffrées avant de quitter votre navigateur. Nos serveurs ne voient jamais vos clés privées brutes ou votre Mot de Passe Principal. Même si notre infrastructure était compromise, les attaquants ne verraient que des données chiffrées.
